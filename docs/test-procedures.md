# テスト手順書

## 日程調整Webアプリ 動作確認手順

---

## 事前準備

1. **環境変数の設定**：`.env` ファイルに全ての変数を設定
2. **DBマイグレーション**：`npx prisma migrate dev --name init`
3. **シードデータ投入**：`npm run db:seed`
4. **開発サーバー起動**：`npm run dev`（http://localhost:3000 でアクセス）

---

## テスト項目

### 【テスト1】ログイン機能

#### 1-1: Microsoft 365 SSOログイン
1. `http://localhost:3000/login` にアクセス
2. 「Microsoft 365 でログイン」ボタンをクリック
3. Microsoftのログイン画面にリダイレクトされることを確認
4. 組織アカウントでログイン
5. ダッシュボード（`/dashboard`）にリダイレクトされることを確認

**確認ポイント**：
- [ ] Microsoftのログイン画面が表示される
- [ ] ログイン後にダッシュボードへ遷移する
- [ ] セッションにユーザー情報が格納される

#### 1-2: アプリアカウントログイン（Credentialsログインは初期パスワードを管理者が設定後にテスト）
1. メールアドレスとパスワードを入力
2. 「ログイン」ボタンをクリック

---

### 【テスト2】日程調整作成

1. ダッシュボードの「新規日程調整」ボタンをクリック
2. **【Step 1】基本情報**を入力：
   - 件名：「テスト打ち合わせ」
   - 時間：60分
   - 場所：「第1会議室」
3. 「次へ」をクリック
4. **【Step 2】参加者**を選択（チェックボックスで1名以上選択）
5. 「次へ：候補日時を確認」をクリック

**確認ポイント**：
- [ ] Microsoft Graph APIが呼び出され、空き時間が取得される（要Microsoft 365環境）
- [ ] 候補日時が一覧表示される（平日9:00〜19:00のスロット）
- [ ] 候補日時が0件の場合は適切なメッセージが表示される

6. **【Step 3】候補日時**：いくつかスロットを選択
7. 「次へ：メール送付」をクリック
8. **【Step 4】メール**：取引先メールアドレスを入力し送信

**確認ポイント**：
- [ ] メールが正常に送信される（受信確認）
- [ ] ダッシュボードに新しい調整が表示される

---

### 【テスト3】社外ゲストの日時選択

1. 送付されたURLをブラウザで開く（`/guest/{token}`）

**確認ポイント**：
- [ ] 認証なしでアクセスできる
- [ ] 候補日時が表示される
- [ ] モバイル（スマートフォン幅）でも正しく表示される

2. 候補日時のひとつを選択
3. 備考メッセージを入力
4. 「この日時で確定する」をクリック

**確認ポイント**：
- [ ] 完了メッセージが表示される（DB設定の文言）
- [ ] 作成者に確定通知メールが送信される
- [ ] 調整のステータスが「確定済み」に変わる

---

### 【テスト4】排他制御（競合テスト）

1. 2つのブラウザタブやデバイスで同じゲストURLを開く
2. 両方で同じ候補日時を選択し、ほぼ同時に「確定」ボタンをクリック
3. 片方は成功し、もう一方はエラーメッセージが表示されることを確認

**確認ポイント**：
- [ ] 競合エラー時に「行き違いで〜」のメッセージが表示される（DB設定の文言）
- [ ] 最新の候補一覧が再表示される
- [ ] 二重予約にならない

---

### 【テスト5】有効期限テスト

1. `.env` の `NEXTAUTH_SECRET` を確認
2. DBで直接 `expires_at` を過去の日時に更新
3. ゲストURLにアクセス

**確認ポイント**：
- [ ] 「URLの有効期限が切れています」と表示される

---

### 【テスト6】管理者設定画面

1. ADMINロールのユーザーでログイン
2. `/admin` にアクセス

**確認ポイント**：
- [ ] 「文言管理」タブで文言を変更して保存できる
- [ ] 「ブランディング」タブで社名・ロゴURLを設定できる
- [ ] 「メールテンプレート」タブでテンプレートを変更できる
- [ ] 「メンバー管理」タブで「Microsoft 365と同期」ボタンが動作する

3. 一般ユーザーで `/admin` にアクセス

**確認ポイント**：
- [ ] ダッシュボードにリダイレクトされる（権限エラー）

---

### 【テスト7】文言管理の反映確認

1. 管理者で「候補日時が0件の場合のメッセージ」を変更・保存
2. 候補日時が0件のゲストURLにアクセス

**確認ポイント**：
- [ ] 変更した文言が社外向け画面に反映されている

---

### 【テスト8】履歴機能

1. 一般ユーザーで `/history` にアクセス

**確認ポイント**：
- [ ] 自分が作成した調整のみ表示される

2. 管理者で `/history` にアクセス

**確認ポイント**：
- [ ] 全メンバーの調整が表示される

3. 「キャンセル」ボタンで調整をキャンセル

**確認ポイント**：
- [ ] ステータスが「キャンセル済み」に変わる

---

## 本番環境（Vercel）での追加確認

- [ ] Vercel環境変数に全ての値が設定されている
- [ ] カスタムドメイン/本番URLでAzure ADのリダイレクトURIが設定されている
- [ ] Supabase本番DBへの接続が正常
- [ ] Vercel Cron（毎日9時）が正常に動作している（Vercel Logsで確認）
- [ ] メール送信がVercelから正常に動作している

---

## パフォーマンス確認

- [ ] ゲストURLのページロードが1秒以内（キャッシュヒット時）
- [ ] キャッシュ未存在時のGraph API呼び出しが10秒以内（Vercelの制限）
